Agentic User Intelligence Harness

Multi-layer agentic architecture for user onboarding, intent understanding, resource search, and adaptive response generation

Hermes Agent · DSPy + RAG + Memory · Self-improving conversational loop

Onboarding Pipeline

  • New-user detection triggers profile wizard
  • Guided questions extract wants, goals, constraints
  • Preference vector initialized from first-session signals
  • Returning users resume with full context preload

Intent Understanding

  • DSPy classifier: fine-tuned on domain-specific categories
  • NER extracts structured fields from free-form input
  • Context enricher merges session + long-term history
  • Router dispatches: search / create / clarify / chitchat

Resource Search & Retrieval

  • Query planner decomposes multi-intent questions
  • Hybrid search: vector similarity + keyword + KG traversal
  • External tools: web, MCP APIs, arXiv, Polymarket, etc.
  • Cross-encoder re-ranker fuses relevance + diversity

Response Generation

  • LLM receives intent + context bundle + user prefs
  • Safety layer: content filter, PII scrubbing, validation
  • Humanizer strips AI-isms, matches user communication style
  • Evaluator scores output via reward model + self-critique

Memory & Learning

  • Short-term: rolling conversation buffer per session
  • Long-term: PostgreSQL + vector store + memory graph
  • Preference learner tracks implicit signals & decay
  • DSPy optimizer updates prompts from reward feedback loop

Continuous Improvement

  • User feedback (thumbs, corrections, ratings) collected
  • Evaluator logs to W&B for experiment tracking
  • DSPy compile() optimizes signatures + teleprompters
  • Self-improving: better prompts → better responses → better prefs
Key Design Principles: The feedback loop is the engine — every interaction refines the user model and the prompting strategy. DSPy serves as the orchestration backbone, treating prompts as programs that can be compiled, tested, and optimized. All user data flows through a privacy-first pipeline with PII scrubbing at the safety layer and encrypted storage in PostgreSQL. The modular architecture allows swapping any component (e.g., replacing the vector store or LLM backend) without breaking the outer loop.
