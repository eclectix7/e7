# Hermes Agent — Tools & Skills Reference

> Generated by Kaizen2. Complete catalog of built-in tools and loadable skills for the Hermes Agent CLI.

## Built-in Tools

### Browser (playwright-based web interaction)

- **browser_navigate** — Load a URL in the headless browser. Use for interactive pages (SPAs, forms, JS-heavy sites). For plaintext endpoints (raw JSON, .md, API responses), prefer `terminal` with curl instead.
- **browser_snapshot** — Get a text accessibility-tree snapshot of the current page. Compact mode shows interactive elements with ref IDs; full mode dumps entire page text.
- **browser_click** — Click an element by its ref ID (e.g. `@e5` from snapshot). For buttons, links, toggles.
- **browser_type** — Type text into an input field by ref ID. Clears the field first.
- **browser_press** — Press a keyboard key (Enter, Tab, Escape, ArrowDown). For submitting forms or keyboard navigation.
- **browser_scroll** — Scroll the page up or down to reveal lazy-loaded or below-fold content.
- **browser_vision** — Take a screenshot and analyze it with vision AI. Use for CAPTCHAs, visual verification, complex layouts, or when the text snapshot misses visual context.
- **browser_console** — Read browser console output / JS errors, or evaluate arbitrary JavaScript in the page context. Use for debugging silent JS errors, inspecting DOM state, extracting data programmatically.
- **browser_get_images** — List all images on the page with URLs and alt text. Useful before calling `vision_analyze` on specific images.

### File System

- **read_file** — Read a text file with line numbers. Supports offset/limit pagination. The go-to instead of `cat`/`head`/`tail`.
- **write_file** — Create or overwrite a file entirely. Creates parent directories automatically. Runs post-write syntax checks on .py/.json/.yaml/.toml. The go-to instead of `echo`/`cat heredoc`.
- **search_files** — Ripgrep-backed search. Two modes: `target="content"` (regex search inside files) and `target="files"` (glob file discovery, sorted by mtime). The go-to instead of `grep`/`rg`/`find`/`ls`.
- **patch** — Targeted find-and-replace with fuzzy matching (9 strategies). Use instead of `sed`/`awk`. Supports single-file replace mode and multi-file V4A patch mode.

### Terminal / Process

- **terminal** — Execute shell commands. Foreground (blocking with timeout) or background with lifecycle tracking. Use for builds, installs, git, scripts, network, package managers — anything that needs a real shell.
- **process** — Manage background processes: list, poll output, view full log, wait for completion, kill, send stdin, close stdin. Use after `terminal(background=true)`.

### Memory & Session

- **memory** — Save durable facts across sessions (user preferences, environment details, project conventions, learned quirks). Injected every turn. Compact, declarative, no task-progress logs.
- **session_search** — FTS5-powered search of past conversation sessions. Three shapes: discovery (query by keyword), scroll (read inside a session around a message), browse (recent sessions list). Use instead of asking the user to repeat themselves.

### Agent Orchestration

- **delegate_task** — Spawn subagents in isolated contexts for parallel work. Use for reasoning-heavy subtasks, tasks that would flood your context with intermediate data, or parallel independent workstreams (research, debugging, code review). Supports batch mode (up to 3 concurrent).
- **mixture_of_agents** — Route a hard problem through 4 reference models + 1 aggregator for collaborative reasoning. Use sparingly for complex math, advanced algorithms, multi-step analytical reasoning.
- **execute_code** — Run Python scripts that can call Hermes tools programmatically. Use when you need 3+ tool calls with processing logic, conditional branching, or loops between them.

### Scheduling

- **cronjob** — Schedule recurring or one-shot tasks. Supports: skill-attached jobs, per-job model overrides, data-collection scripts, script-only "watchdog" mode, job chaining (context_from), toolset restriction, profile isolation, working directory injection, no_agent mode (script output delivered verbatim).

### Skills System

- **skill_view** — Load a skill's full SKILL.md content and list its linked files (references, templates, scripts). Always call this before using a skill — skills contain exact commands, workflows, and pitfalls.
- **skill_manage** — Create, update (patch/edit), or delete skills. Skills are procedural memory: reusable approaches for recurring task types. Patch immediately when you discover missing steps or pitfalls during use.
- **skills_list** — List all available skills with descriptions and optional category filter.

### Messaging / Communication

- **send_message** — Send messages to connected platforms (Telegram, Discord, Slack, Signal, Matrix, Yuanbao). Supports channels, threads, DMs. Use `action='list'` first to discover available targets.
- **clarify** — Ask the user a question when you need clarification, feedback, or a decision. Supports multiple-choice (up to 4 options) or open-ended mode. Not for yes/no on dangerous commands (terminal handles that).

### Vision & Media

- **vision_analyze** — Load an image (URL, local path, or data URL) into context so the agent can see it. For vision-capable models the pixels go directly into context; for non-vision models an auxiliary vision model provides a text description.
- **text_to_speech** — Convert text to speech audio. Output saved as .mp3. Provider and voice are user-configured.

### Task Management

- **todo** — Manage an in-session task list (create, update, reorder, mark complete/cancelled). One item `in_progress` at a time. Use for complex tasks with 3+ steps.

### MCP (Model Context Protocol)

- **mcp_context7_resolve_library_id** — Resolve a library/package name to a Context7-compatible library ID. Must call this before `mcp_context7_query_docs` unless the user provides an explicit library ID.
- **mcp_context7_query_docs** — Retrieve up-to-date documentation, code examples, and API details from Context7 for any programming library or framework. Use for current docs on e.g. Next.js, Express, React, Supabase, etc.

## Skills (Loadable Procedural Knowledge)

Skills are loaded with `skill_view(name)` and provide step-by-step workflows, exact commands, and known pitfalls. Organised by category.

### Autonomous AI Agents

- **claude-code** — Delegate coding tasks to Claude Code CLI (features, PRs). Steps for setup, prompt engineering, handoff patterns.
- **codex** — Delegate coding to OpenAI Codex CLI (features, PRs). Installation, usage patterns, error handling.
- **hermes-agent** — Configure, extend, or contribute to Hermes Agent itself. Exact commands for `hermes config set`, tools, providers, models, plugins, voice, gateway, setup.
- **kanban-codex-lane** — Use when a Hermes Kanban worker wants to run Codex CLI as part of a pipeline.
- **opencode** — Delegate coding to OpenCode CLI (features, PR review). Setup, usage, review workflows.
