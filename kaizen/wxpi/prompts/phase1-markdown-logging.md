# Cursor Prompt — Phase 1: Markdown Logging

## Context

Wxpi uses markdown files as a human-readable audit trail for all context operations. The database schema and capture module are already created. Now we need the markdown file management layer.

## Requirements

Create a Markdown Logger module that:

1. **Directory Initialization** (`src/markdown/file-manager.ts`):
   - On app startup, ensure the following directory structure exists under the app's document directory:
     ```
     wxpi-data/
     ├── context/
     │   └── daily/
     ├── improvements/
     ├── summaries/
     │   ├── weekly/
     │   └── monthly/
     └── user-profile.md
     ```
   - Uses `expo-file-system` for all file operations
   - Creates directories recursively if they don't exist
   - Creates `user-profile.md` with a default template if it doesn't exist

2. **Daily Context Log** (`src/markdown/daily-log.ts`):
   - Function `appendSignal(date: string, signal: NormalizedSignal): Promise<void>`
   - Appends to `context/daily/YYYY-MM-DD.md`
   - Format each entry as: `- **HH:MM** [signalType] source: summarizedPayload`
   - If file doesn't exist for that date, create it with header: `# Context Log — YYYY-MM-DD\n\n## Signals\n`
   - Summarize payload to a single line (max 100 chars) — don't dump full JSON
   - Use `FileSystem.appendAsStringAsync()` for atomic appends

3. **Improvement Log** (`src/markdown/improvement-log.ts`):
   - Function `logImprovement(date: string, improvement: ImprovementAction): Promise<void>`
   - Appends to `improvements/YYYY-MM-DD.md`
   - Format with timestamp, trigger type, analysis summary, actions taken, and HITL status
   - If file doesn't exist, create with header: `# Improvement Log — YYYY-MM-DD\n`

4. **User Profile Markdown** (`src/markdown/user-profile.ts`):
   - Function `updateUserProfile(profile: UserProfileData): Promise<void>`
   - Overwrites `user-profile.md` with current user data
   - Format as a structured markdown document with sections: Identity, Technical Profile, Preferences, Interests, Context Stats
   - Called by the self-improvement engine after applying changes

5. **Weekly Summary** (`src/markdown/weekly-summary.ts`):
   - Function `writeWeeklySummary(weekStart: string, content: string): Promise<void>`
   - Writes to `summaries/weekly/YYYY-WNN.md`
   - Content is generated by the self-improvement engine (Phase 4)

6. **File Rotation** (`src/markdown/rotation.ts`):
   - Function `rotateOldFiles(maxAgeDays: number = 30): Promise<void>`
   - Deletes daily context logs older than maxAgeDays
   - Never deletes improvement logs, summaries, or user-profile.md
   - Called during self-improvement cycle

## File Structure

```
src/markdown/
├── file-manager.ts     # Directory initialization
├── daily-log.ts        # Daily context log appending
├── improvement-log.ts  # Improvement action logging
├── user-profile.ts     # User profile markdown management
├── weekly-summary.ts   # Weekly summary writing
├── rotation.ts         # File cleanup
├── types.ts            # Shared types
└── index.ts            # Public API
```

## Constraints

- Use `expo-file-system` (not react-native-fs)
- All file operations must be async
- Handle file-not-found errors gracefully (create if missing)
- Never block the UI thread
- Use atomic append operations (appendAsStringAsync) to avoid corruption
- Include proper error handling — markdown writes are non-critical (system works without them)

## Verification

After implementation, verify:
1. Directory structure is created on app startup
2. Daily log file is created and appended to correctly
3. Improvement log entries are formatted properly
4. User profile markdown is generated with correct structure
5. File rotation deletes only daily logs older than threshold
6. Concurrent appends don't corrupt files
